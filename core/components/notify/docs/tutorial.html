Overview

Notify will let you send an email notification to Subscribers on your site (or to all active users) when a page is created or updated (or just when you feel like sending them a message). Rather than sending the whole page, Notify sends a message with a link to the new or updated page. Optionally, you can specify tags for the message and only subscribers with those tags set will receive the email. Notify works in conjunction with the Subscribe extra, which lets users register at your site and manage their preferences (tags).

Installing Notify for MODX Revolution

Go to System | Package Management on the main menu in the MODX Manager and click on the "Download Extras" button. That will take you to the Revolution Repository (AKA Web Transport Facility). Put Notify in the search box and press Enter. Click on the "Download" button, and when it changes to "Downloaded," click on the button that takes you back to Package Management. Click on the "Install" button next to Notify in the grid. The Notify package should now be installed.

During the installation, the Notify TVs will be connected to the site's default template. If you need them attached to other templates, you'll have to connect them manually by editing each of the TVs, checking the boxes on the "Template Access" tab, and clicking on the "Save" button.

If you want Notify to post to Twitter and/or Facebook, you'll need to set the four Notify System Settings for Twitter (see below).

When Notify is first installed and you check the Template Variable tab for a resource, the Notify TVs may not be filled in. If so, clear the site cache and reload the page.

Using Notify

The Notify plugin pre-fills the Notify TVs for a page when you edit it in the Create/Edit Resource panel. The values for the email, email subject, and Tweet come from the Notify Tpl chunks. You can change the text in any of the text TVs before sending an update, but be sure to save the page before sending.

When sending updates, Notify acts when you preview the page but only when you have set the Notify options and saved the page. Unless you set the options and save the page first, you'll see the regular page preview. To send an update, just check the appropriate radio buttons, save the page, then preview it. Important! It's critical that you remember to save the page before previewing or Notify will use the old settings for the TVs and your update may not be sent.

The various options are independent, so you can send yourself a test email, send a message to subscribers, or send a Tweet without activating the other options.

Notify pre-fills the email subject, email message, and Tweet fields for you, but you can put whatever you want into them. If you just want to email your subscribers to tell them what you had for breakfast, you can do that. The only page-specific information will be the URL (filled in by Notify) and any Resource Content tags that you include in the Tpl chunk (e.g. pagetitle, introtext, description , etc.). You're free to add, or delete any of them. In other words, you can use Notify to send any email message to your subscribers, optionally selected by Group or Tag. All resource fields are available through placeholder tags (e.g., [[+pagetitle]]) as well as [[+url]] and [[++site_name]]

Groups

Email is only sent to active users. The Groups TV can contain a comma-separated list (no spaces) of User Groups to send to. You can specify them by name or by User Group ID number. By default, Notify sends to members of the "Subscribers" User group, but you can change that by editing the "nf_groups" TV and changing the default value. If you leave the Groups TV blank, the email message will be sent to all active users on your site.

Tags

The Tags TV is blank by default, so the email is sent to all members of User Groups specified in the Groups TV. If you enter a comma-separated list of tags in the Tags TV, only users who have those tags (interests) will receive the email. You can fill in the user interests for each user as a comma-separated list (no spaces) in the "Comment" field of the User Profile (or optionally in an extended field of the User Profile), but the Subscribe package will do that for you and let users manage their own interest lists. See the Subscribe package documentation for more information on how the Tags are stored.

If you're not using the Subscribe package and you don't want to use the "Comment" field to store the user's interests, you can create two new System Settings (put them in the Notify namespace to make them easier to find):


sbs_use_comment_field   (set the value to 'No')
sbs_extended_field   (name of the extended field you want to use)

You'll have to create the extended field for each user.

Email Log

For each email update, Notify writes a log file to the core/components/notify/notify-logs directory. The file is named for the date and time of the update. The log shows the message sent, the users and their tags, and if possible any problems in sending the email to each user.

Posting Updates to Twitter

In order for Notify to post updates to Twitter, you'll need to create a Twitter App at the Twitter Dev site (if you don't have one already). Follow the instructions at that URL for creating a new application. Once you have it all set up, you can go to the the URL above and select the "My Applications" breadcrumb at the top left. Then, click on your Twitter username and select the "OAuth Tool" tab. you should see the four keys you need to set the Notify System Settings.

To set the System Settings, go to System | System Settings in the MODX Manager and select the "notify" Namespace in the drop-down list at the top of the grid (it will show the "core" Namespace to begin with). Posting Updates to Facebook

Notify does not, at present, post directly to Facebook, but you can set up an account at Selective Twitter in about 30 seconds. Once you've done that, simply adding the hashtag #fb to any Tweet and the Tweet will be posted to your Facebook page in addition to Twitter. If you set the ¬ifyFacebook property, the #fb hashtag will be added automatically to the end of the Tweet and you'll see it in the Notify form. The Facebook App will remove that hashtag in the Tweet.

You can also use the Twitter App if you want to post *all* your Tweets to Facebook, but I don't recommend doing that if you're a heavy Twitter user because it will load up your Facebook page with lots of Tweets that most visitors won't want to see.
